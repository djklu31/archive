PHP Implementation:

Getting Started

Before we start:

Now, we will delve into the PHP portion of our implementation. Firstly, we will handle
what happens when the "Submit" button is hit. Conceptually, for the PHP implementation
that will be presented, we want to think about handling a few features before we start coding.
For tab generation, we want to give feedback to the user after the submit button is hit
to let them know to wait a few moments. This is because it takes a few seconds XML to
be parsed when there are potentially hundreds of tab entries (eg. The Beatles). We will
go into this XML retrieval and processing later on. For this example, when the submit
button is hit, I want the artist, song, tab version, and tab rating (out of 5) to be
printed out. In addition, I want to have a button next to every tab selection where
the user could have the tab pop-up in a new window when clicked.

Sessions:

The first thing that I did when implementing the PHP portion was to start a new session.
The sole reason for this is because I didn't want to pass a bunch of parameters from moving
from function to function. Storing key-value pairs in the SESSION array allows me
to reference values from any function (similar to global variables without the downsides).
This is entirely optional as there are other ways of passing parameters and saving/retrieving values
throughout the code. The way I started my session was by simply declaring: session_start(); in the PHP section.
In my code, I added this even before the HTML search for portion:

<?php
session_start();
?>
